# DocuChef 라이브러리 작성 지침

## 기본 원칙
- 문서 템플릿 처리를 위한 통합 라이브러리로서 다양한 문서 형식(Excel, PowerPoint, Word)을 일관된 방식으로 지원
- Excel은 ClosedXML.Report.XLCustom, PowerPoint와 Word는 DollarSignEngine 기반으로 템플릿 처리 위임
- 라이브러리 사용자 경험을 일관되게 유지하면서 각 문서 형식의 특성에 맞는 고유 기능 제공
- 간결한 API와 풍부한 기능 간의 균형 유지

## 코드 구조 및 설계
- **모듈화 설계**: 각 문서 타입별로 분리된 네임스페이스와 클래스 구조 유지
- **공통 인터페이스**: `IRecipe`, `RecipeBase` 등 공통 기반 클래스 활용하여 일관성 보장
- **부분 클래스 활용**: 500줄 이상의 클래스는 `{ClassName}.{Feature}.cs` 형식으로 분할
- **확장성 고려**: 향후 추가될 문서 형식이나 기능을 쉽게 통합할 수 있는 구조 설계

## 코드 작성 스타일
- **간결성**: 불필요한 코드를 제거하고 핵심 기능에 집중
- **명확한 명명 규칙**: 직관적인 클래스, 메소드, 변수명 사용
- **일관된 포맷팅**: 표준 C# 코딩 규칙 준수
- **영문 작성**: 모든 코드와 주석은 영어로 작성

## 오류 처리 및 로깅
- **예외 계층**: `DocuChefException`을 기본으로 하는 명확한 예외 계층 구조 정의
- **상세한 오류 메시지**: 사용자가 문제를 쉽게 진단할 수 있는 상세 메시지 제공
- **중앙화된 로깅**: `Logger` 클래스를 통한 일관된 로깅 메커니즘 사용

## 문서화 및 주석
- **XML 주석**: 공개 API에는 `<summary>` 태그로만 간결한 설명 제공
- **자체 문서화 코드**: 의미 있는 이름과 명확한 구조로 코드 자체가 문서화 역할
- **예제 코드**: 복잡한 기능에는 주석에 간단한 사용 예시 포함

## 테스트 및 품질 관리
- **단위 테스트**: 핵심 기능에 대한 포괄적인 단위 테스트 구현
- **통합 테스트**: 실제 문서 생성 시나리오를 검증하는 통합 테스트 작성
- **성능 고려**: 대용량 템플릿 처리 시 성능 저하가 없도록 설계

## 외부 라이브러리 활용
- **ClosedXML.Report**: Excel 문서 처리에 활용, 불필요한 기능 재구현 지양
- **DollarSignEngine**: PowerPoint와 Word 문서의 표현식 처리에 활용
- **OpenXML SDK**: 저수준 문서 조작에 활용, 필요한 경우에만 직접 사용

## 구현 세부 사항
- **템플릿 처리 위임**: 각 엔진의 특성에 맞게 데이터 준비 및 위임에 집중
- **확장성 확보**: 사용자 정의 함수, 변수 등을 쉽게 추가할 수 있는 구조 제공
- **엣지 케이스 처리**: 다양한 템플릿 구조와 데이터 형식에 대응할 수 있는 안정적 처리

## 유지보수 및 진화
